你是如何使用浏览器并实现模拟点击的

13:07:22 ASSISTANT

我使用浏览器并实现模拟点击主要通过 两种并行机制：一种是深层的协议级控制（CDP），另一种是表层的系统级模拟（xdotool）。

以下是我的实现逻辑详解：

1. 协议级控制：浏览器自动化 (CDP)

这是我处理网页任务的首选方式。我通过 Chrome DevTools Protocol (CDP) 直接与 Chromium 引擎对话。

  • 核心工具：browser 专用工具。
  • 实现流程：
    1. 观察（Snapshot）：我先对网页发送 snapshot 指令，它会返回页面的 ARIA 树（无障碍树）。我会看到类似 [e12] button "登录" 的标识。
    2. 定位：系统会自动计算该元素在页面上的坐标或 DOM 路径。
    3. 动作（Act）：我发送 browser(action=act, request={kind: 'click', ref: 'e12'})。
  • 优点：极其精准，不受窗口缩放或遮挡影响，甚至可以操作隐藏在 Shadow DOM 里的元素。

2. 系统级模拟：桌面交互 (OS Level)

如果任务涉及浏览器窗口本身（如点击地址栏、关闭标签页）或者是非浏览器应用，我会切换到 Linux 系统的输入模拟工具。

  • 核心工具：xdotool 和 wmctrl（在 TOOLS.md 中有记录）。

  • 模拟点击命令：

    # 将鼠标移动到坐标 (500, 300) 并执行左键点击
xdotool mousemove 500 300 click 1

  • 窗口定位：
    我先通过 wmctrl -l 获取浏览器窗口的 ID，然后用 xdotool windowactivate <ID> 将其置顶。

  • 实现流程：

    1. 截图：使用 scrot 或 canvas(action=snapshot) 获取当前桌面图像。
    2. 视觉分析：通过多模态视觉模型识别出按钮的 (X, Y) 坐标。
    3. 物理点击：调用 xdotool 执行点击动作。

3. 我是如何“看见”并“瞄准”的？

我并不是盲目点击，我的操作逻辑是：

  1. 解析结构：优先读取网页的 HTML/无障碍标签。
  2. 视觉校准：如果标签不清晰，我会通过视觉模型分析截图。
  3. 坐标转换：将网页内的相对坐标转换为 Linux 桌面系统的绝对像素坐标。

4. 实际操作演示 (以点击搜索框为例)

如果我要在网页上点击一个搜索框并输入文字，我的内部指令流通常如下：

// 1. 获取页面快照，找到搜索框的引用 ID (假设为 e5)
browser(action="snapshot")

// 2. 执行点击和输入
browser(action="act", request={
  kind: "type",
  ref: "e5",
  text: "OpenClaw 架构图"
})

// 3. 模拟按下回车键
browser(action="act", request={
  kind: "press",
  key: "Enter"
})

总结

  • 网页内操作：使用 browser 工具走 CDP 协议（逻辑点击）。
  • 网页外/桌面操作：使用 xdotool 模拟鼠标硬件信号（物理点击）。

这种“双持”架构让我既能像开发者一样操控 DOM，也能像真人一样操作整个 Linux 桌面。

最快捷的“整机移植”方案（推荐）

如果你觉得手动安装上述依赖太麻烦，最稳妥的移植方式是使用 Docker 镜像同步：

  1. 导出当前镜像（如果你做了自定义修改）：

    docker commit <容器ID> my-openclaw-custom:latest
docker save my-openclaw-custom:latest | gzip > openclaw_image.tar.gz

  2. 在新服务器导入：

    docker load < openclaw_image.tar.gz

  3. 挂载 Workspace：
    启动时挂载你打包的 workspace 目录。这样，你不仅移植了浏览器工具，还移植了所有的登录状态和操作记忆。

为什么不能只复制一个脚本？

因为 browser 工具依赖于：

  • CDP 端口：Chromium 必须以 --remote-debugging-port=9223 模式启动。
  • 屏幕坐标：点击位置依赖于 Xvfb 虚拟屏幕的分辨率设置。
  • 权限：模拟硬件点击需要对 /dev/uinput 或 X 服务的访问权限。

总结：
要移植这个技能，镜像 (Image) 是容器，配置 (Workspace) 是灵魂。确保新服务器运行相同的 Docker 镜像并挂载了你的 Workspace，这项技能就会完美复现。
